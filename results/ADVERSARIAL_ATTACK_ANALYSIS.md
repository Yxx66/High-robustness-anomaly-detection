# 对抗攻击效果差异分析报告

## 🎯 核心问题：为什么四个数据集的对抗攻击效果不明显？

---

## 📊 对抗攻击效果对比

| 数据集 | 基线准确率 | 对抗攻击后准确率 | 准确率下降 | 攻击效果 |
|--------|-----------|----------------|-----------|---------|
| **CICIDS-2017** | 95.94% | **7.60%** | **-88.34%** ❌ | ✅ **显著** |
| **CICIDS-2018** | 99.50% | **99.45%** | **-0.05%** | ❌ **无效** |
| **CICIDS-2019** | 97.68% | **97.49%** | **-0.19%** | ❌ **无效** |
| **BoTIoT-2018** | 100.00% | **96.62%** | **-3.38%** | ⚠️ **轻微** |
| **N-BaiIoT-2021** | 99.94% | **80.48%** | **-19.46%** | ⚠️ **中等** |

---

## 🔍 五大关键原因分析

### 1️⃣ **数据集类别不平衡问题**

#### CICIDS-2017（平衡，攻击有效）
```
BENIGN: 5,791 (11%)
ATTACK: 47,344 (89%)
平衡度: 1:8.2
```
- ✅ 相对平衡，模型需要学习复杂决策边界
- ✅ 对抗扰动能有效混淆模型

#### CICIDS-2018（极度不平衡，攻击无效）
```
Benign: 56,556 (95%)
ATTACK: 3,009 (5%)
平衡度: 1:18.8
```
- ❌ **95%正常流量**：模型学到的是"默认预测为正常"的简单规则
- ❌ FGSM扰动不足以让正常样本跨越极宽的决策边界
- ❌ 模型对少数攻击类过拟合，对扰动鲁棒

#### CICIDS-2019（极度不平衡，攻击无效）
```
BENIGN: 1,455 (3%)
ATTACK: 55,448 (97%)
平衡度: 1:38.1
```
- ❌ **97%攻击流量**：模型学到的是"默认预测为攻击"的简单规则
- ❌ 无论如何扰动，样本仍在攻击区域内
- ❌ 单一Portmap攻击特征过于明显

---

### 2️⃣ **攻击类型复杂度**

| 数据集 | 攻击类型数量 | 攻击复杂度 | FGSM效果 |
|--------|-------------|-----------|---------|
| **CICIDS-2017** | **9种** | 高：Bot, DDoS, DoS(4种), FTP-Patator, Heartbleed, Infiltration | ✅ 有效 |
| **CICIDS-2018** | **2种** | 低：DoS GoldenEye, DoS Slowloris | ❌ 无效 |
| **CICIDS-2019** | **1种** | 极低：仅Portmap | ❌ 无效 |
| **BoTIoT-2018** | **5种** | 中：TCP, UDP, Service_Scan, OS_Fingerprint, HTTP | ⚠️ 轻微 |
| **N-BaiIoT-2021** | **多种** | 中高：多种IoT僵尸网络攻击 | ⚠️ 中等 |

**原理**：
- ✅ **CICIDS-2017多样性高**：9种攻击类型 → 决策边界复杂 → 易被FGSM扰动攻破
- ❌ **CICIDS-2018/2019单一性高**：1-2种攻击 → 决策边界简单 → FGSM扰动不足以跨越边界

---

### 3️⃣ **特征区分度问题**

#### CICIDS-2017（特征区分度适中）
```python
特征数: 78
攻击特征与正常特征有适度重叠
→ 模型依赖特征组合进行分类
→ FGSM扰动能改变特征组合 → 攻击成功
```

#### CICIDS-2018（特征区分度过高）
```python
DoS攻击特征极其明显:
- 流量速率异常（GoldenEye/Slowloris特征极端）
- 与正常流量完全分离
→ epsilon=0.3的扰动不足以跨越巨大的特征差距
→ 攻击失败
```

#### CICIDS-2019（特征区分度过高）
```python
Portmap攻击特征单一且明显:
- 端口111的特征极其显著
- 二分类问题过于简单
→ 模型学到的是简单的阈值规则
→ FGSM扰动无法改变核心特征值
→ 攻击失败
```

### 4️⃣ **FGSM超参数epsilon=0.3的局限性**

#### 当前配置
```python
# pipeline.py
epsilon = 0.3  # 固定值
perturbations = epsilon * tf.sign(gradients)
```

#### 问题分析
| 数据集 | epsilon=0.3 | 建议epsilon | 原因 |
|--------|------------|------------|------|
| **CICIDS-2017** | ✅ 合适 | 0.1-0.5 | 类间距离适中 |
| **CICIDS-2018** | ❌ 太小 | **0.8-2.0** | 类间距离极大，需要更大扰动 |
| **CICIDS-2019** | ❌ 太小 | **1.0-3.0** | 特征差异巨大，需要极大扰动 |
| **BoTIoT-2018** | ⚠️ 偏小 | 0.5-1.0 | 完美性能，需要更强攻击 |
| **N-BaiIoT-2021** | ⚠️ 偏小 | 0.4-0.8 | 高维特征(115)，需要更大扰动 |

#### 数学原理
```
对抗样本: x_adv = x + ε·sign(∇_x L(θ, x, y))

情况1（CICIDS-2017）：
- 类间距离: d ≈ 5-10
- ε = 0.3 ≈ 3-6% × d → 足以跨越边界 ✅

情况2（CICIDS-2018）：
- 类间距离: d ≈ 50-100 (DoS极端)
- ε = 0.3 ≈ 0.3-0.6% × d → 远不足以跨越边界 ❌

情况3（CICIDS-2019）：
- 类间距离: d ≈ 100+ (Portmap单一特征)
- ε = 0.3 ≈ 0.3% × d → 完全不足以跨越边界 ❌
```

### 5️⃣ **模型学习策略差异**

#### CICIDS-2017（复杂学习）
```python
模型学习策略:
if (feature1 > t1) AND (feature2 < t2) AND ...:
    → ATTACK
else:
    → BENIGN

特点:
- 多特征组合决策
- 决策边界复杂、非线性
- 对扰动敏感 ✅
```

#### CICIDS-2018（简单学习）
```python
模型学习策略:
if (flow_rate > 极大阈值):  # DoS特征
    → ATTACK
else:
    → BENIGN (默认，95%概率)

特点:
- 单特征阈值决策
- 决策边界极其简单
- 对小扰动不敏感 ❌
```

#### CICIDS-2019（极简学习）
```python
模型学习策略:
if (port == 111 OR traffic_pattern == Portmap):
    → ATTACK (默认，97%概率)
else:
    → BENIGN

特点:
- 几乎无条件预测ATTACK
- 决策边界退化为常数
- 对扰动完全不敏感 ❌
```

```python
def adaptive_epsilon(dataset_name):
    epsilon_map = {
        'CICIDS-2017': 0.3,      # 保持原值
        'CICIDS-2018': 1.5,      # 增大5倍
        'CICIDS-2019': 2.0,      # 增大6.7倍
```

## 📋 总结

### 核心结论
1. **CICIDS-2017是唯一有效的对抗攻击评估数据集** 🏆
2. **CICIDS-2018/2019对FGSM攻击"免疫"** ❌
3. **原因**: 极度不平衡 + 攻击类型单一 + epsilon太小
4. **N-BaiIoT-2021次之**，BoTIoT-2018攻击效果最弱

### 数据集选择建议
| 研究目的 | 推荐数据集 | 原因 |
|---------|-----------|------|
| 对抗攻击评估 | **CICIDS-2017** 🏆 | 攻击效果显著(-88%) |
| 对抗训练评估 | **CICIDS-2017** 🏆 | 防御提升显著(+92%) |
| 基线性能测试 | **BoTIoT-2018** | 完美性能(100%) |
| IoT场景评估 | **N-BaiIoT-2021** | 高维特征(115) |
| ❌ 不推荐 | CICIDS-2018/2019 | 对抗攻击无效 |

### 改进优先级
1. 🔴 **高优先级**: 为不同数据集使用自适应epsilon
2. 🟡 **中优先级**: 尝试PGD等更强攻击方法
3. 🟢 **低优先级**: 数据重采样或特征归一化改进
